## 0.1.0 (unreleased)

- Illumina `assemble` through preprocess, *k*-mer counting, trusted DBG, tip + bounded diamond bubble simplification, unitigs + reference contigs, FASTA/GFA export (**GFA 1.0** `H`/`S`/`L` when applicable, **`P`** when a contig path matches a single unitig path); optional rayon-backed *k*-mer sort; FASTA ingest; CLI output paths; CI on MSRV + stable + **Phase-1 benchmark gate**; `ARCHITECTURE.md`; `fuzz/` **cargo-fuzz** harness (`parse_fastq`).
- Checkpoints: preprocess, counts, simplified **DBG** (`graph/`), stitched **unitigs + contigs** (`export/sequences.json`), strict per-stage manifests; resume reloads graph and export when valid; graph rewrite clears stale `export/`; pure **Kâ‚‚,â‚‚** diamonds skip automatic collapse.
- CLI: `--config` TOML merge (CLI overrides file; `k` may come from file alone); stdout **`-`** for FASTA/GFA paths; `--no-resume` / `--no-strict-checkpoints`; simplification flags + `[assemble.simplify]` TOML.
- Benchmark gate: `scripts/benchmark_gate.sh` (reference-free stats + **minimap2** or substring reference check); `fixtures/tiny_ref.fa`; `tools/manifest.toml` pins **minimap2**; committed golden assembly under `fixtures/expected/ref_free_smoke/` verified by `ref_free_smoke.sh`.
- `CONTEXT.md`: bounded **Phase-2 Illumina** glossary (exports, mates, benchmarks, CLI, versioning, counting, simplification, checkpoints, observability, phasing ladder, reproducibility, preprocess, Tier-1, contigs, scaffolds, FASTA/GFA, benchmark automation/fixtures, error/CLI, fuzzing, CI cadence) plus **Phase-2 deferral** and **Phase promotion policy** cross-links.
- **Phase-2 Illumina benchmark gate** automation: `scripts/phase2_illumina_benchmark_gate.sh` (runs `benchmark_gate.sh`, diploid reference layer, graph summaries, haplotype metrics; optional `TREX_RUN_QUAST=1` + `reference_quast.sh`); synthetic two-parent fixture under `fixtures/phase2_synthetic/` with SHA-256 digests in `tools/manifest.toml`; CI runs fast smoke on every PR and **minimap2** reference smoke on `main`/`master`, **tags**, **schedule**, and **workflow_dispatch** only. `ARCHITECTURE.md` notes contract-vs-code boundary.
- **Phase-2 Illumina diploid (experimental)**: `--diploid` / `[assemble.diploid]` with optional `insert_mean_bp` / `insert_stddev_bp`; near-balanced diamond bubbles (within 5% branch support) are not collapsed; GFA `H` line carries `XX:Z:trex-phase2-illumina` when diploid is on; graph checkpoints record diploid resume identity so toggling options rebuilds the graph safely; optional diploid **GFA `L`** unitig adjacency links when emitted.
- **Phase-2 Illumina (additional experimental)**: trusted *k*-mer multiplicity **primary FASTA** collapse; conservative **mate-pair** edge boost on existing DBG edges (paired + insert prior + `preprocess/pair_layout.json` for resume); multi-segment **`P`** when contig vertex paths concatenate full unitigs; mirror **`p2hâ€¦`** **`P`** rows with `TS:Z:trex-unphased-hap-mirror`; graph checkpoint field **`phase2_mate_bridge_v1`**; scripts `phase2_illumina_haplotype_metrics.sh`, `reference_quast.sh` (opt-in `TREX_RUN_QUAST=1`), `pr_smoke.sh`, `tools/benchmark_matrix.toml`, ADR `docs/adr/0001-phase2-deferrals-and-primary-collapse.md`.
- Unitig **stitch_sequence**: multi-read unitig paths try both orientations of the first *k*-mer and each next canonical vertex, keeping the lexicographically smallest valid full sequence (extends **Phase-1 contig tie-break** spirit for stitched unitigs); unit tests in `trex::dbg::unitig`.
- Governance loop: Rust `xtask` validators/benchmark runner, `docs/CAPABILITIES.md`, and ADR `docs/adr/0003-benchmark-tiers-and-ci-contract.md` make benchmark rows, CI tiers, scripts, operator capability claims, and benchmark artifacts fail-fast in CI; `xtask gate --tier pr` now owns the PR orchestration.
- Benchmark matrix: first governed real-reference micro row (`fixtures/phix174/`, RefSeq `NC_001422.1`) with digest-pinned deterministic reads, direct release `trex` execution, observed Trex counters, exact candidate *k*-mer count, FASTQ/FASTA/GFA metrics, wall time, max RSS, and artifact sizes in `target/benchmarks/*.json`.
- Biological benchmark data: `tools/benchmark_data.toml` plus `xtask fetch-data` add external ENA rows for E. coli MG1655 SRR001666 and true diploid S. cerevisiae BTT / ERR1308583, preparing bounded ignored `data/` subsets with pinned SHA-256s and source md5/provenance metadata.
- Real-read correctness/performance: canonical DBG walks now skip unorientable candidate paths instead of aborting assembly, clearing the yeast BTT diploid biological row; first graph-walk optimization cuts the PhiX174 benchmark from 26.65 s to 18.34 s with unchanged output metrics.
- Sparse-read correctness: tip clipping now preserves isolated low-coverage linear components instead of peeling them to nothing, clearing the E. coli MG1655 biological row from 0 contigs to 1,979 contigs with default settings.
- Biological reference quality: `xtask fetch-data` now prepares pinned public reference FASTAs, direct Trex benchmark JSON reports reference k-mer containment, `TREX_RUN_QUAST=1` runs row-local QUAST / MetaQUAST, and the governed E. coli matrix includes a 10k-pair scale-up row.
- Larger biological profiling rungs: governed manual rows now cover E. coli MG1655 SRR001666 at 100k read pairs and S. cerevisiae BTT / ERR1308583 diploid at 10k read pairs, with prepared FASTQ SHA-256s pinned in the data catalog and manifest.
- Graph-walk scaling: linear/cyclic connected components now use one deterministic traversal plus reverse-orientation scoring instead of greedy walks from every vertex; E. coli 10k output metrics stayed identical while wall time dropped from 6.61 s to 5.67 s.
- Literature archive: `literature/` now tracks assembler papers, archived public PDFs, source-only entries, and a review queue for translating OLC/DBG, long-read, diploid/T2T, metagenome, polishing, and evaluation work into Trex gates.
- Literature-informed framework: `docs/ASSEMBLER_FRAMEWORK.md` and `tools/assembler_framework.toml` now define validated modules for the evidence ledger, graph IR, simplification policy, path/scaffold builder, assembly quality, and deferred graph adapters; `xtask validate-framework` is included in the default validator.
- Simplification observability: tip clipping, diamond resolution, diploid balanced-diamond retention, and ambiguous **Kâ‚‚,â‚‚** skips now return current-run decision counters through `AssembleResult` and CLI tracing without changing graph behavior.
- Phase-2 mate bridge observability: existing-edge mate boosting now records typed current-run evidence in `trex::evidence`, `AssembleResult`, `evidence.json`, tracing, and `xtask bench` JSON without changing graph behavior.
- Graph annotations: simplified DBG nodes and unitigs now emit copy-number class, repeat-suspicion, and min/mean/max multiplicity summaries through `AssembleResult`, `annotations.json`, CLI tracing, and `xtask bench` JSON without changing FASTA/GFA output.
- Simplification policy: tip clipping and diamond-bubble handling now flow through decision records with actions, reasons, affected nodes, scores, and retained/skipped motifs; default graph mutations and golden outputs are unchanged, and `simplification.json` is included in direct benchmark JSON.
- Scaffold artifact: Phase-2 mate evidence now emits deterministic bridge candidates and explicit-overlap scaffold paths in `scaffolds.json` and direct benchmark JSON without changing `contigs.fa` or adding mate-derived DBG edges.
- Multi-*k* graph selection: explicit `--kmer-ladder` / `k_ladder` mode now scores independent candidate graphs, selects one graph for the normal assembly path, writes `multi_k.json`, rejects checkpoint reuse until selected-*k* checkpoint identity is implemented, and leaves the one-*k* default unchanged.
- Compact graph walk adapter: `DbgGraph` can now produce a dense internal node-id view, and reference contig walking uses that view for component traversal, edge weights, and multiplicity tie-breaks while keeping byte-string graph names, checkpoints, FASTA, and GFA output unchanged.
- Post-assembly audit: Trex now emits `audit.json` and `audit.tsv` with low read-support regions, mate-pair disagreement hints when mate evidence exists, and collapsed-repeat suspicion derived from graph annotations; the audit is report-only and does not change FASTA.
- Diploid graph semantics: `--parent1-reference` / `--parent2-reference` and `[assemble.diploid]` parent reference keys now emit `diploid.json` parent-specific k-mer evidence and GFA path tags while preserving the single primary FASTA contract and explicitly avoiding full haplotype FASTA claims.
- Benchmark quality artifacts: read-vs-assembly k-mer quality now reports total-weighted assembly-only k-mer breadth alongside the existing distinct-kmer fraction and approximate QV fields.
- Benchmark diploid metrics: direct Trex benchmark rows can declare parental references and emit per-parent reference k-mer quality; the synthetic Phase-2 row now produces this JSON on nightly/manual tiers.
- Development protocol: worker agents now have explicit read-only and implementation modes; implementation workers may edit only assigned files in orchestrator-created temporary worktrees, while `git`, `cargo`, formatting, and gates remain orchestrator-only.
- Graph-walk scoring now carries greedy candidate path scores during construction instead of recomputing edge-weight sums during candidate selection, preserving tie-break behavior while reducing repeated adjacency lookups.
- Gzip ingest now uses multi-member decoding, so concatenated `.gz` FASTQ/FASTA streams are read as one logical input instead of stopping after the first gzip member.
